Gauge strip — design decisions
Layout
Icon centered above, bar below. Each gauge reads as a unit — icon gives instant identity, bar shows state.
Bar height
7px — substantial enough to read state changes clearly. Not decorative, genuinely informative.
Strip height
~72px total with padding. Proportional presence at top without dominating the reading area.
Colours
⚡ amber · 🍺 orange-red · 🌿 green · 🍔 yellow. Story-defined via CSS variables.
Kiff hidden
Score gauge not shown during play — revealed only at end-of-game summary screen.
Long story block
Choices appear at end of scroll
The story block can be any length. The player scrolls through the full prose before reaching the choices — choices are always at the end of the content, never floating above unread text. Shown here at two mid-game gauges states for context.
↕
Scroll inside the phone. The story text requires scrolling to reach the choices below. This is intentional — the player must read the full paragraph before the decision appears. On short blocks (Tab 1), choices are immediately visible. On long blocks, they're at the bottom of the scroll.

Scroll behaviour — design intent
Short blocks
Choices visible immediately below the prose — no scroll needed. Natural for simple decision moments.
Long blocks
Player scrolls through the full story block before reaching choices. Choices are in the document flow — never floating above unread content.
Implementation
Single scroll area: prose + separator + choice cards all in flow. Gauge strip is the only fixed element (top). No sticky choice footer.
Gauge during scroll
Always visible at top, fixed. Player can check gauge state at any point while reading. Acts as a persistent status layer above the content.
Bold / italic
Supported inline within the story block — sourced from JSON config. No block-level formatting (headings, lists). Prose only.
