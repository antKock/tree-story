# Story 3.5: Character Sheet & Inventory

Status: done

## Story

As a player,
I want to view my full character state (all gauges, stats, and inventory) on demand without leaving the story,
so that I can check where I stand without being pulled out of the reading experience.

## Acceptance Criteria

1. Opens as a shadcn `Sheet` overlay — it overlays the story screen, it does not replace it
2. Shows ALL gauges including the score gauge (Kiff) as fill bars — no raw numbers (FR6)
3. Shows all player stats with their allocated values using Inter typography
4. Shows the current inventory as a list of item names — empty state shows a neutral message if no items (FR7)
5. Tapping outside the sheet or a close control dismisses it and returns to the story
6. Opening and closing the sheet triggers no engine state changes or localStorage writes
7. CharacterSheet is triggered exclusively by tapping anywhere on the gauge strip — no other trigger points

## Dev Notes

- **shadcn Sheet component:** shadcn provides a `Sheet` component that renders as a panel overlay. Use `side="bottom"` for a slide-up panel on mobile (feels natural for a character sheet). The shadcn Sheet handles the backdrop click to close via `onOpenChange`.
- **Opening does not trigger engine:** This is critical. The character sheet is purely presentational — it reads `engineState` but never writes to it. No `resolveChoice`, no `applyDecay`, no `persistence.save` when opening or closing.
- **All gauges (no filter):** The GaugeStrip hides `isHidden` gauges, but the CharacterSheet shows everything. The player can see Kiff/score in the CharacterSheet. This is intentional — it's a "check your detailed status" screen.
- **Item names in inventory:** `engineState.inventory` is `string[]` of item IDs. Ideally, map these to display names via story config. If story config has an `items` dictionary, look up `config.items[itemId]?.name || itemId`. If no items dictionary, display the ID directly (still readable for Dub Camp).
- **Mobile bottom sheet:** On mobile, `side="bottom"` creates a sheet that slides up from the bottom. This is the most natural mobile gesture for a character info panel. The shadcn Sheet handles the slide animation.
